# 2026-07-24 v1.2.0 hermes-agent sync — Cursor

## 概要

セマンティックバージョンを **v1.1.0 → v1.2.0** に上げ、`zapabob/hermes-agent` の `scripts/windows/watchdog-go` から未取り込みの高価値ヘルパーを additive merge。multi-platform `tar.gz` Release・GH Pages・README を更新。

## 背景・要求

- 本リポが canonical（hermes-agent は READ ONLY）
- P1–P6（state machine / heartbeat / IPC / warm-start / Job Object / update suppress）を壊さない
- v1.1.0 で取り込み済みの `--skip-build` / auth-drift は維持し、残ギャップのみ移植

## 適用 SOP / Skill

- Implementation Start Gate / MILSPEC Codex Standard / SOP-Go
- Project `AGENTS.md`（Build / ForceRestart / dist gitignore）

## 前提・判断（CoT）

1. Diff 結果: 本リポの `watchdog.go` / `server.go` は既に P1–P6 で hermes-agent より大きい
2. 残ギャップは主に `process_windows.go`（netstat fallback / bounded taskkill）と Start/Build タイムアウト、async prewarm
3. `findHealthyDesktopBackend` の Ready/Live probe は本リポ側が優るため上書きしない
4. async Prewarm∥RunLoop は hermes-agent と同パターン；`BackendManager.mu` で spawn は直列化されるが state 読み書きの軽微 race は残留リスクとして記録

## 変更対象ファイル

- `process_windows.go` — `listeningPIDsOnPort`、netstat fallback、bounded `stopProcessPID`、WMI 8s timeout
- `process_windows_test.go` — invalid port 回帰
- `main.go` — async prewarm（RunLoop と並列）
- `health.go` — `testBackendAuth` timeout 3s（hermes-agent 整合）
- `scripts/Build-HermesGoWatchdog.ps1` — SkipTest 時は tidy もスキップ
- `scripts/Start-HermesGoWatchdog.ps1` — `BuildTimeoutSec` / `RunBuildTests` / bounded build
- `scripts/Package-Release.ps1` — default `v1.2.0`
- `docs/*`, `README.md` — Pages / CTA / checksums
- `_docs/2026-07-24_v1.2.0-hermes-agent-sync_Cursor.md`（本ログ）

## 実装詳細

### hermes-agent から merge したもの

| Item | Source | Notes |
|------|--------|-------|
| `listeningPIDsOnPort` | process_windows.go | netstat LISTENING PID 列挙 |
| `stopListenersOnPort` netstat fallback | process_windows.go | WMI timeout 時も squatters を置換 |
| bounded `stopProcessPID` (5s) | process_windows.go | wedged taskkill で probe が固まらない |
| WMI timeout messaging (8s) | process_windows.go | Hermes.exe / full scan |
| async Prewarm ∥ RunLoop | main.go | cold serve が監視ループを止めない |
| SkipTest → skip tidy | Build-*.ps1 | operator 再起動経路 |
| BuildTimeoutSec / RunBuildTests | Start-*.ps1 | タイムアウト付きビルド |
| auth probe 3s | health.go | 既存 `testBackendAuth` 調整 |

### 意図的に上書きしなかったもの

- 単純な `watchdog.go` / `server.go`（本リポの P1–P6 が正）
- `findHealthyDesktopBackend` の Live/Ready probe（本リポ優位）
- hermes-agent ネスト layout のパス解決（本リポは standalone）
