# 2026-07-24 v1.1.0 multi-platform release — Cursor

## 概要

セマンティックバージョンを **v1.1.0** に上げ、Windows バイナリ + Linux/macOS ソース（stub 付き）の multi-platform `tar.gz` を GitHub Release に公開。GH Pages の version badge / ダウンロード CTA / アーキテクチャ図を更新。

## 背景・要求

- v1.0.0 → v1.1.0（minor: auth-drift / `--skip-build` / packaging 拡張）
- Windows: `hermes-watchdog.exe` + scripts
- Linux/macOS: source + Setup scripts（optional cross binary、フル Desktop 監視は主張しない）
- `SHA256SUMS` + Pages CTA → Release assets
- hermes-agent には触れない

## 適用 SOP / Skill

- Implementation Start Gate / MILSPEC Codex Standard / SOP-Go
- Project `AGENTS.md`（Build script / dist gitignore / Windows-first）

## 前提・判断

- フルプロセス監視は Windows-only（`process_windows.go` / Job Object / Named Pipe）
- 非 Windows は最小 stub（`process_stub.go` / `backend_stub.go` / `process_alive_stub.go`）で `go build`/`go test` が通る程度
- `processAlive` は `tasklist` 依存をやめ `OpenProcess` に変更（ハング回避）
- WMI クエリにタイムアウトを追加（マシン上の WMI 競合でテストが固まる問題）

## 変更対象ファイル

- `backend.go`, `backend_test.go`, `health.go`, `process_windows.go`, `go.mod`（既存 migration 取り込み）
- `process_common.go`, `process_alive_windows.go`, `process_alive_stub.go`, `process_stub.go`, `backend_stub.go`（新規）
- `watchdog.go`（processAlive 分離）
- `scripts/Setup-HermesGoWatchdog.ps1`, `scripts/Setup-HermesGoWatchdog.sh`, `scripts/Package-Release.ps1`
- `docs/*`（Pages v1.1.0 / diagram / checksums）
- `README.md`, `AGENTS.md`
- `_docs/2026-07-24_v1.1.0-multiplatform-release_Cursor.md`（本ログ）

## 実装詳細

1. auth-drift / `--skip-build` migration を本リポに確定
2. 非 Windows stub + portable helpers 抽出
3. Windows exe / Linux amd64 / Darwin amd64+arm64 を cross-build（Darwin 再ビルド中にディスク不足 → 直前の darwin 成果物を梱包）
4. `Package-Release.ps1` で 5 アセット + `SHA256SUMS`
5. Pages: neon gold 維持、Windows/Linux/macOS CTA、architecture JPG

## 実行コマンド

```powershell
go test -count=1 -p 1 -timeout 180s .
go build -buildvcs=false -trimpath -ldflags "-s -w" -o dist\hermes-watchdog.exe .
# GOOS/GOARCH cross builds
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\Package-Release.ps1 -Version v1.1.0
gh release create v1.1.0 ...
git push origin main
```

## 残留リスク

- Linux/macOS バイナリは stub 動作のみ；誤って本番監視に使うと Desktop 再起動が動かない
- Darwin バイナリは trim 再ビルド失敗前の成果物（機能的には同等 stub 付き）
- GitHub Pages 反映に数分遅延ありうる
- ディスク空きが逼迫している環境では cross-build が失敗しうる

## 次の推奨アクション

- ディスク掃除後に Darwin trim 再ビルドしてアセット差し替え（任意）
- CI で `Package-Release.ps1` を自動化
- Pages モバイル幅の目視確認
