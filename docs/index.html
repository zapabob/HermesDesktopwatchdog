Hermes Go Watchdog

Mutual supervision for Desktop and Serve — operator-only, loopback-first.

A standalone Windows Go process that keeps Hermes Desktop and hermes serve alive with health checks, warm-start, Job Objects, and update suppress — not a plugin, MCP, or skill.

Architecture

Desktop client, Serve backend, and Tailnet enclave under one mutual supervision loop.

Capabilities

Built for operators who need a single restart authority without exposing admin APIs unguarded.

Mutual health

Loopback health, heartbeat ingest, and report-only IPC so Desktop/Backend cannot restart themselves.

Warm start

Ordered warm-restart sequencer with Job Object containment for the managed backend process tree.

Update suppress

Env, lockfile, and API hooks pause aggressive recovery while an update is in progress.

Shipped phases P1–P6

Lifecycle manager path from state machine through update suppress.

P1ServiceState machine + crash-loop RestartPolicy — watchdog is sole restart authority.
P2Mutual health (/health, /api/status, backend live/ready) + heartbeat ingest.
P3Report-only IPC (Named Pipe + HTTP /api/v1/report|/command|/ipc).
P4Warm-start sequencer (warm_restart) — see WARM-START-CONTRACT.
P5Windows Job Object for managed backend; renderer anomaly policy stub.
P6Update suppress via env, update.lock, and POST /api/v1/update-suppress.

Install

Unpack the release archive, set an admin token, then start with the helper script from source — or run the exe directly.

# PowerShell
tar -xzf hermes-watchdog-windows-amd64-v1.0.0.tar.gz
$env:HERMES_WATCHDOG_ADMIN_TOKEN = "your-secure-operator-token"
.\hermes-watchdog.exe
# Default listen: 127.0.0.1:9920

Secrets stay in env vars. Never commit tokens or auth keys. Full operator notes live in the repo under _docs/OPERATOR.md.
